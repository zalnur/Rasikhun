Solution

Problem

The Selection interface is small, but its implementation is scattered: membership, cache identity, target Group filtering, and Comparison Pool lists live in different modules.

Solution

Let the Selection module own Selection identity and derived Location lists; callers stop rebuilding the same facts.

Benefits

  • locality: one Selection rule
  • leverage: shared cache keys
  • fewer drift points
  • tests use public seam
